# 定稿命令

当用户调用此命令时，帮助用户将当前工作定稿并提交到 Git 仓库。

## 执行步骤

1. **检查当前状态**：运行 `git status` 查看当前有哪些更改

2. **确认定稿内容**：向用户展示将要定稿的文件列表，询问用户确认

3. **获取版本信息**：
   - 如果用户提供了参数 `$ARGUMENTS`，则使用该参数作为版本描述
   - 如果没有提供参数，询问用户本次定稿的版本描述（如：v1.0、初稿、终稿等）

4. **执行定稿**：
   - 运行 `git add .` 添加所有更改
   - 运行 `git commit -m "定稿: <版本描述>"` 提交更改
   - 询问用户是否需要创建 Git tag 标记此版本

5. **推送到远程**：
   - 运行 `git push` 推送提交
   - 如果创建了 tag，同时运行 `git push --tags` 推送标签

6. **完成确认**：显示提交成功信息，包含 commit hash 和推送状态

## 注意事项
- 如果没有任何更改，提示用户当前没有需要定稿的内容
- 提交信息格式统一为：`定稿: <版本描述>`
- Tag 格式统一为：`release/<版本号>` 或用户自定义
